This page contains plain-language definitions for the most common terms you will encounter while using version control.

**Clone**

> The act of downloading a complete copy of an online project folder from the GitHub website down to your local computer's physical drive.

**Command line**

> The text-only program on your computer used to interact with operating systems and tools by typing written instructions rather than clicking visual buttons.

**Commit**

> An official, permanent snapshot of your tracked project files saved directly into your computer's history log. Think of it like a milestone save point in a game.

**Git**

> The local software engine that runs directly on your computer to track file modifications and power your project's personal time machine.

**GitHub**

> The online website used to host copies of your project folders online so you can safely back up your work, share files via web links, or collaborate with teammates.

**Project folder (Repository)**

> A regular folder on your computer that has been given a history tracking log so that version control tools can actively monitor changes made inside it.

**Push**

> The command used to upload your local saved project snapshots up to your online account on the GitHub website to update your online backup.

**Staging area**

> The intermediate preparation environment where you gather specific file changes into a conceptual packing box before officially locking them into your timeline with a commit.
